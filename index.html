Adaptability Lab: Cloud and Autonomic Computing

The general purpose of this lab was to enhance a knowledge of concepts and technologies for virtualization techniques. These techniques are used in IT environments that support the provisioning (i.e. development, deployment, management) of novel software systems (with their potential constraints such as QoS and security). These environments are typically dynamic and distributed.

The objectives were:

  • Discover virtualization archetectures and technologies
  • Recognize the fundamental differences between the two main types of virtualisation hosts, i.e. virtual machines (VM) and containers (CT).
  • Operate VirtualBox VMs provisioning, in NAT mode, and make appropriate configurations so that VMs can access and be accessed through the Internet.
  • Operate Docker containers provisioning, and make appropriate configurations so that containers can access and be accessed through the Internet.
  • Operate proper VMs provisioning with OpenStack and manage the networking connectivity.
  • Implement a simple orchestration tool in order to provision a Web 2-tier application using node.js services

OM2M Platform

In order to monitor the sensors we set up an OM2M Platform which have all the ressources needed. This platform has the following tree archetecture of ressources:

We organized the elements by platforms AE (application entity). In each platform there are the sensors and actuators containers. Each sensor/actuator contains its data CIN.

Sensor Manufacturing at AIME

The goals of this training unit were :

  • Firstly, manufacture a gas sensor at AIME.
  • Secondly, eximine the sensor in order to find its characterizations.
  • Finally, build a datasheet related to these characterizations to allow a user to understand how to use our sensor.
Sensor manufacturing

The Gas Sensor we created is based on tungsten trioxide nanoparticles. Small combs silicon substrates fill with tungsten trioxide nanoparticles (WO3) deposit react to the ambient concentration of gas.

To do that there were three steps:

  • Synthesis of nanoparticles of tungsten

    • We followed a protocol to synthesize tungsten trioxide molecules. It was a very intresting protocol insofar as it allowed us to understand how import the security was in such as chimical experience. Moreover, we understood how import is the ecological impact, and how to reduce it by limiting the waste. Finaly, it was important to be very meticulous in such as experiences.

  • Photolithography: Conception of the combs

    • The purpose of this part is to create the sensor chip which will contain the nanoparticles that we synthesized before. The next figure shows the chip archetecture:

  • DiElectrophoresis: integration of tungsten in the combs

    • For the integration of the nanoparticles into the sensor we used DiElectrophoresis method. This part will allow the resistors to be well electrically connected. In this method, we connected the pins of the comb, then deposit a nanoparticle solution on the sensor. We passed an electric current for one minute. Our particles will, thanks to the dielectrophoretic force, integrate on the comb. Once the DiElectrophoresis done we saw the tungsten particles disposed in the microscope:

Analog Electronic Labs

After we manufactured the sensor, we had to integrate it in an electronic circuit. The aim of this circuit is to amplify the out signal from the gas sensor in order to be used by an arduino. In fact, the arduino's ADC can not treat the the out signal while this signal is not amplified: the entry impedance of the Arduino is much lower than the impedance of the sensor.

LTSpice: How we simulated the circuit

We used the software LTSpice to simulate this analog circuit. It allows us to build schematics we want and then run some simulations.

The next figure shows the final circuit that we used. Firstly, we simulated the sensor by a low pass filter. Then, we placed it at the entry of the amplifier that can amplify it and obtain the wanted signal. Finally, we had to filter the noise, to do that, we used several filters.

Kicad: Integration on the PCB

When we were sure that this circuit was the perfect one for our sensor we integrated it in our final PCB on kicad. The following figure shows the Kicad shematics:

Microcontrollers & Open Source Hardware

After we manufactured the sensor, did the integration in an electronic circuit, the purpose of this Microcontrollers & Open Source Hardware is to realise a connected gas sensor. The system had to be able to send data over the web with Lora technology and had to be plug with a special shield on an Arduino. The main part of this project were:

  • Firstly, Create a custom Arduino Shield for the whole system
  • Secondly, Send data to internet through a Lora radio
  • Finaly, Exploit received data with Node-Red

Read the sensor data and send them through a LoRa radio

Once our sensor manufactured and we got the its PCB for Arduino, we can now monitor its data. In fact, we used an Arduino with a LoRa radio module (the RN2483) the build this connected gas sensor.
